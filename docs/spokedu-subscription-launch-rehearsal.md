# 스포키듀 구독 출시 리허설 (MVP)

목적: 출시 전에 운영팀이 실제 대응 가능한지 검증한다.

---

## 1) 리허설 시나리오

### 시나리오 A: 신규 가입 후 체험 시작
- 기대 결과
  - 센터 생성 성공
  - 구독 상태 `trialing`
  - 체험 종료일 표시

### 시나리오 B: 체험 만료 전환
- 기대 결과
  - `trialing -> expired`
  - 플랜 `free` 전환
  - 반 한도 1개 적용

### 시나리오 C: 결제 지연 복구
- 기대 결과
  - `active -> past_due` 전환 공지
  - 확인 후 `past_due -> active` 복구

### 시나리오 D: 해지 요청 대응
- 기대 결과
  - 운영 승인 후 `canceled` 반영
  - 사용자 안내 템플릿 발송

---

## 2) 실행 체크리스트

- [ ] 운영 담당자 2인 배정 (실행/검증 분리)
- [ ] 테스트 대상 센터 ID 확보
- [ ] SQL 접근 권한 확인
- [ ] 안내 템플릿 사전 준비
- [ ] 변경 이력 기록 채널 준비 (노션/슬랙/이슈)

## 4) 1차 리허설 반영 사항 (2026-04-03)

- 반영 1: `/billing` 데드 링크 제거를 위해 구독 화면으로 리다이렉트 경로를 추가했다.
- 반영 2: 플랜 가격/혜택 하드코딩 중복을 줄이고 공용 카탈로그 기반으로 통일했다.
- 반영 3: 설정 화면 상태 라벨을 한글화해 운영 문의 시 혼선을 줄였다.
- 반영 4: 자동 결제 미연동 사실을 화면 문구에 명시해 기대치 불일치를 줄였다.

---

## 5) 출시 승인 기준

- 4개 시나리오 모두 pass
- 상태 전환 처리 시간 30분 이내
- 사용자 안내 템플릿 즉시 발송 가능
