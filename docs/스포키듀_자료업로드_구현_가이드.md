# 스포키듀 구독 — 자료 업로드 구현 가이드

스크린 플레이 72개(하드코딩 + 엑셀)와 프로그램 144개(엑셀 → DB)의 구체적인 코드 작업 순서·파일·API·컴포넌트 정리.

---

## Part 1. 스크린 플레이 72개 (하드코딩 + 엑셀)

### 1.1 목표

- 72개 목록은 **코드/정적 데이터**로 유지(DB 없음).
- 운영은 **엑셀**로 72개 편집 후, CSV 내보내기 → JSON/TS 변환 → 해당 파일만 갱신.

### 1.2 타입·형식 정의

**파일**: `app/lib/spokedu-pro/screenplayCatalog.ts` (또는 `types/screenplay.ts`에 타입만 분리)

```ts
// 한 항목 필드 (엑셀 컬럼과 1:1)
export interface ScreenplayCatalogItem {
  id: string | number;       // 1~72 또는 코드
  modeId: string;            // CHALLENGE | FLOW | 반응인지 | 순차기억 | 스트룹 | 이중과제
  title: string;
  subtitle?: string;
  description?: string;
  sortOrder: number;
  presetRef?: string;        // 예: 2026-03-W2
  thumbnailUrl?: string;
}

export type ScreenplayCatalog = ScreenplayCatalogItem[];
```

- `modeId`는 실행 시 `onOpenInteractive(modeId)`에 그대로 넘기므로, 기존 모드 ID와 동일한 값 사용.

### 1.3 카탈로그 데이터 위치 (택 1)

| 옵션 | 경로 | 용도 |
|------|------|------|
| A. TS 상수 | `app/lib/spokedu-pro/screenplayCatalog.ts` | `export const SCREENPLAY_CATALOG: ScreenplayCatalog = [ ... ]` |
| B. 정적 JSON | `public/data/screenplay-catalog.json` | `fetch('/data/screenplay-catalog.json')` 로 로드 |

- **A**: 빌드에 포함, 타입/자동완성 좋음.  
- **B**: JSON만 교체하면 되므로 엑셀→JSON 스크립트와 연동하기 쉬움.  
- 추천: 초기에는 **A(TS 상수)** 로 1~2개 샘플 넣고, 72개 확정 후 **B(JSON)** 로 옮기거나, 스크립트로 JSON → TS 생성해도 됨.

### 1.4 스크립트: CSV → JSON/TS (엑셀 연동)

**파일**: `scripts/csv-to-screenplay-catalog.js` (또는 `.ts` + ts-node)

- **입력**: CSV 경로 (예: `./screenplay-72.csv`).  
  - 헤더: `id,mode_id,title,subtitle,description,sort_order,preset_ref,thumbnail_url`
- **출력**:
  - 옵션 1: `public/data/screenplay-catalog.json` 생성.
  - 옵션 2: `app/lib/spokedu-pro/screenplayCatalog.generated.ts` 에 `export const SCREENPLAY_CATALOG = [ ... ]` 문자열 생성.
- **동작**: CSV 파싱 → 행별로 `ScreenplayCatalogItem` 형태 객체로 변환 → `sort_order` 기준 정렬 → 파일 쓰기.
- **실행 예**: `node scripts/csv-to-screenplay-catalog.js ./screenplay-72.csv`

(필요 시 엑셀 템플릿용 CSV 샘플도 `docs/screenplay-72-template.csv` 로 두면 됨.)

### 1.5 구독자 화면 수정

**파일**: `app/(pro)/spokedu-pro/views/ScreenplayView.tsx`

- **현재**: `MODES` 배열(6개)을 map 해서 카드 렌더링.
- **변경**:
  - 카탈로그 소스: `SCREENPLAY_CATALOG` import 또는 `fetch('/data/screenplay-catalog.json')` 결과 사용 (캐시/상태는 훅으로).
  - 카드 그리드: 72개 `ScreenplayCatalogItem` 기준으로 렌더링. `title`, `subtitle`, `description`, `modeId` 사용.
  - 클릭 시: 기존과 동일하게 `onOpenInteractive(item.modeId)` 호출 (필요하면 `presetRef` 등 추가 인자 전달).
- **모드 메타**(아이콘, 그라디언트 등): 72개 항목에 `modeId`가 있으므로, `modeId` → 메타 매핑 객체를 한 곳에 두고 카드에서 참조.

### 1.6 작업 순서 (스크린 플레이)

1. `ScreenplayCatalogItem` 타입 정의 (`app/lib/spokedu-pro/screenplayCatalog.ts` 또는 types 파일).
2. `SCREENPLAY_CATALOG` 상수 또는 `public/data/screenplay-catalog.json` 추가 (샘플 1~2개).
3. `ScreenplayView.tsx`가 카탈로그를 읽어 72개 카드로 표시하도록 수정.
4. (선택) `scripts/csv-to-screenplay-catalog.js` 추가 및 `docs/screenplay-72-template.csv` 템플릿 추가.
5. 72개 확정 후 엑셀 → CSV → 스크립트 실행으로 카탈로그 파일 갱신.

## Part 2. 프로그램 144개 (엑셀 → DB)

### 2.1 목표

- 144개는 **DB** (`spokedu_pro_programs`) 에 저장.
- Admin에서 **CSV/엑셀 업로드** → 검증 → 반영(upsert).

### 2.2 DB 사용 전환

**파일**: `app/api/spokedu-pro/programs/route.ts`

- **현재**: `const DB_READY = false` → 목 데이터 반환.
- **변경**: `DB_READY = true` 로 전환 (또는 `process.env.SPOKEDU_PRO_PROGRAMS_DB_READY === 'true'` 등 환경 변수로 제어).
- **GET**: 이미 DB 분기 코드 있음. `spokedu_pro_programs` 에서 `is_published = true` 만 조회하도록 유지.

### 2.3 일괄 임포트 API

**파일**: `app/api/spokedu-pro/programs/import/route.ts` (신규)

- **메서드**: `POST`
- **인증**: `requireAdmin` 사용 (`@/app/lib/server/adminAuth`).
- **요청**: `multipart/form-data` 로 `file` (CSV 또는 .xlsx).  
  - 또는 `Content-Type: application/json` 으로 `{ rows: ProgramRow[] }` 도 가능 (Admin이 클라이언트에서 파싱 후 전송).
- **동작**:
  1. 파일 파싱 (csv-parse 또는 xlsx 라이브러리). UTF-8 처리.
  2. 행별로 `title`, `theme_key`, `theme_label`, `role`, `category`, `video_url`, `description`, `gradient_class` 등 매핑.
  3. `theme_key`, `category` 허용 값 검증. 실패 행은 오류 목록에 담기.
  4. `spokedu_pro_programs` 에 upsert (id 있으면 update, 없으면 insert) 또는 “이번 업로드로 전부 교체” 정책 중 하나로 일괄 처리. `getServiceSupabase()` 사용.
  5. 응답: `{ success: number, failed: number, errors?: { row: number, message: string }[] }`.
- **스키마**: `spokedu_pro_programs` 컬럼과 맞추기. (마이그레이션 `20260308000000_spokedu_pro_commercial.sql` 참고.)

### 2.4 CSV/엑셀 컬럼 (프로그램) — 구독자 모달(SpokeduProDrawer)과 동일

엑셀 칸과 컬럼은 **프로그램 상세 모달(제목, 영상, Target Brain, Physical, 준비물, 셋업 가이드, Expert Tip)** 과 동일하게 맞춰 두었습니다.

| 칸 | 컬럼명 | 필수 | 설명 |
|----|--------|------|------|
| A | title | ○ | 프로그램 제목 |
| B | theme_key | ○ | 테마 영문 키 (intro, co-op, speed-reaction, cognitive, challenge, tag-duel, variant-sports) |
| C | theme_label | ○ | 선생님 화면에 보여질 한글 테마 이름 |
| D | role | ○ | 프로그램 역할 |
| E | category | ○ | Play / Think / Grow |
| F | video_url | | 유튜브 영상 주소 |
| G | description | | 프로그램 상세 (모달의 Expert Tip) |
| H | gradient_class | | 카드 배경색상 코드 (선택, 비워두면 기본값) |
| I | id | | 고유 식별 번호 (선택, 있으면 해당 행 수정) |
| J | target_brain | | Target Brain (모달과 동일) |
| K | target_physic | | Physical (모달과 동일) |
| L | tool | | 준비물 (모달과 동일) |
| M | setup_guide | | 셋업 가이드 (모달과 동일) |

### 2.5 Admin 업로드 UI

**위치**: Admin 스포키듀 프로 편집 영역.  
- **옵션 1**: `app/admin/spokedu-pro/page.tsx` 상단 또는 사이드에 “자료 업로드” 링크 → 별도 페이지.  
- **옵션 2**: `app/admin/spokedu-pro/upload/page.tsx` (신규) — “프로그램 CSV 업로드” 전용.

**페이지 동작**:

1. 파일 선택 (input type=file, accept=".csv,.xlsx").
2. (클라이언트 또는 서버) 파싱 후 미리보기 테이블 (처음 10행 + “외 N건”).
3. “검증” 버튼: API에 검증만 하는 엔드포인트가 있으면 호출, 없으면 임포트 API가 검증+반영 한 번에.
4. “반영” 버튼: `POST /api/spokedu-pro/programs/import` 호출 → 결과 토스트 (N건 성공, M건 실패 + 실패 행 번호).
5. (선택) “템플릿 다운로드” 링크: `docs/programs-144-template.csv` 또는 API에서 생성해 제공.

### 2.6 구독자 쪽 프로그램 목록

**파일**: `app/(pro)/spokedu-pro/views/LibraryView.tsx`

- **현재**: `PROGRAM_BANK` (dashboardDefaults.ts 더미) 사용.
- **변경**:  
  - `useEffect` 또는 훅으로 `GET /api/spokedu-pro/programs` 호출 (theme_key, category, q, limit, offset 이미 지원).  
  - 응답 `data`를 프로그램 목록으로 사용.  
  - `PROGRAM_BANK` fallback은 DB 데이터 없을 때만 사용하거나 제거.

### 2.7 작업 순서 (프로그램)

1. `app/api/spokedu-pro/programs/route.ts` 에서 `DB_READY = true` (또는 env) 전환.
2. `app/api/spokedu-pro/programs/import/route.ts` 추가 (파싱, 검증, upsert, 오류 반환).
3. Admin `app/admin/spokedu-pro/upload/page.tsx` (또는 기존 admin 페이지 내) 업로드 UI 추가.
4. `LibraryView.tsx` 가 프로그램 API GET 결과를 쓰도록 수정.
5. (선택) `docs/programs-144-template.csv` 템플릿 추가.

## Part 3. 미리보기 — Admin에서 하나요?

**네. 미리보기는 Admin에서 이루어집니다.** 구분은 아래 두 가지입니다.

### 3.1 업로드 시 미리보기 (반영 전 확인)

- **위치**: Admin 프로그램 업로드 페이지 (`/admin/spokedu-pro/upload` 또는 동일 영역).
- **시점**: CSV/엑셀 파일을 선택한 뒤, **“반영” 버튼을 누르기 전**.
- **내용**:
  - 파싱된 행을 테이블로 표시 (예: 처음 10행 + “외 N건”).
  - 검증 결과가 있으면 행 단위로 “통과 / 오류” 표시.
- **목적**: DB에 넣기 전에 “이 내용이 맞는지” Admin이 확인. 잘못된 행만 수정해 다시 업로드할 수 있음.

### 3.2 구독자 화면 그대로 보기 (발행 결과 확인)

- **위치**: Admin 페이지 상단의 **“구독자 보기 (새 탭)”** 링크 (현재 `app/admin/spokedu-pro/page.tsx` 에 있음).
- **동작**: `/spokedu-pro` 를 새 탭에서 연다. **구독자와 동일한 라우트·동일한 데이터**로 보임.
- **시점**: 프로그램 반영(및 필요 시 Publish) 후, “실제 구독자에게 어떻게 보이는지” 확인할 때.
- **참고**: 스크린 플레이는 정적 파일/코드라 배포만 하면 구독자와 동일. 프로그램은 DB `is_published = true` 인 행만 구독자 API에 나가므로, Admin이 “구독자 보기”로 들어가면 발행된 144개만 보게 됨.

**정리**: “미리보기”는 모두 **Admin 쪽**에서 합니다. (1) 업로드 직후 = 반영 전 테이블 미리보기, (2) 반영 후 = 구독자 보기 링크로 실제 노출 확인.

## Part 4. 구독자에게 보이는 과정 (안내)

구독자가 **스크린 플레이 72개**와 **프로그램 144개**를 보게 되는 흐름을 단계별로 정리합니다.

### 4.1 전제

- 구독자는 **로그인**한 상태로 `/spokedu-pro` 에 접근합니다. (미로그인 시 로그인 유도 또는 리다이렉트.)
- 보이는 데이터는 **이미 발행된 것만** (프로그램은 `is_published = true`, 스크린 플레이는 배포된 카탈로그/정적 파일).

### 4.2 구독자 진입 → 메인 구조

1. **진입**: 브라우저에서 `/spokedu-pro` 접속 (또는 사이드바/메뉴로 “스포키듀 구독” 선택).
2. **레이아웃**: 구독자 전용 레이아웃 (사이드바 숨김 또는 간소화, 전체 화면 대시보드).
3. **화면 구성**: 좌측(또는 상단) 메뉴로 **뷰 전환** — 예: “스크린 플레이 실행”, “100대 프로그램 뱅크”, “로드맵”, “데이터 센터” 등.

### 4.3 스크린 플레이 72개가 보이는 과정

1. 구독자가 메뉴에서 **“스크린 플레이 실행”** (또는 동일 의미 메뉴) 선택.
2. **ScreenplayView** 가 로드됨.
3. **데이터 로드**:  
   - 카탈로그가 **TS 상수**면 빌드 시 포함되어 즉시 사용.  
   - **JSON** 이면 `fetch('/data/screenplay-catalog.json')` 등으로 72개 목록 로드.
4. **화면**: 72개 항목을 **카드 그리드**로 표시 (제목, 부제목, 설명, 모드별 아이콘/색상 등).
5. **클릭**: 카드 클릭 → `onOpenInteractive(modeId)` 호출 → 해당 모드(CHALLENGE, FLOW 등)의 **실행 화면**으로 전환 (전체 화면 인터랙티브).
6. **실행**: 기존 스크린 플레이 엔진이 그대로 동작. 72개는 “선택지 목록”만 제공하고, 실행 로직은 기존 코드 유지.

**요약**: 구독자는 “스크린 플레이 실행” 메뉴 → 72개 카드 목록 → 원하는 카드 클릭 → 실행. **미리보기는 Admin이 “구독자 보기”로 같은 경로 들어가면 동일하게 확인 가능.**

### 4.4 프로그램 144개가 보이는 과정

1. **대시보드(홈)**:  
   - “이번 주 테마” 등 **큐레이션된 행**에 프로그램 일부가 노출될 수 있음 (기존 `dashboard_v4` 등).  
   - 이 행의 프로그램은 **API** `GET /api/spokedu-pro/programs` 또는 공용 콘텐츠에서 가져온 **published** 데이터.

2. **100대 프로그램 뱅크**:  
   - 구독자가 메뉴에서 **“100대 프로그램 뱅크”** (Library) 선택.
3. **LibraryView** 로드.
4. **데이터 로드**:  
   - `GET /api/spokedu-pro/programs` 호출 (필터: theme_key, category, q, limit, offset).  
   - API는 **`is_published = true`** 인 행만 반환. (Admin이 업로드·반영 후 “발행” 처리한 144개.)
5. **화면**: 144개를 **테마/카테고리 탭·검색**으로 필터해 그리드/리스트로 표시. 카드 클릭 시 상세(영상 URL, 설명 등) 노출.
6. **영상**: `video_url` 이 유튜브 URL이면 기존처럼 썸네일·재생 연동.

**요약**: 구독자는 “100대 프로그램 뱅크” 메뉴 → API로 144개(발행된 것만) 수신 → 테마/카테고리/검색으로 탐색 → 카드 클릭 시 상세·영상. **미리보기는 Admin이 “구독자 보기”로 같은 경로에서 확인.**

### 4.5 Draft / Publish 를 쓰는 경우

- 프로그램을 **draft 먼저 반영**하고, 검수 후 **publish** 하는 구조라면:
  - **임포트 API**: 반영은 “draft” 상태로 넣거나, 별도 “발행” API/버튼으로 `is_published = true` 로 전환.
  - **구독자 API**: `GET /api/spokedu-pro/programs` 는 **published 만** 조회하므로, 발행 전에는 구독자에게 **노출되지 않음**.
  - Admin은 “구독자 보기” 전에 **draft 미리보기**를 Admin 전용 화면에서 볼 수 있게 할 수 있음 (선택).

### 4.6 한 줄 요약

- **미리보기**: Admin에서 (1) 업로드 직후 반영 전 테이블 미리보기, (2) “구독자 보기” 링크로 실제 노출 확인.
- **구독자 과정**: 로그인 → `/spokedu-pro` → “스크린 플레이 실행”에서 72개 카드 → 클릭 시 실행 / “100대 프로그램 뱅크”에서 144개(발행분) 조회·탐색 → 카드 클릭 시 상세·영상.
