# 스포키듀 구독 상용화 체크리스트

**목적**: 구독의 핵심 가치 = **주간 수업 자료 제공**. 그 다음에 코치 시간 절감·리포트·리텐션.

---

## 1. 가치 우선순위 (명확히 두기)

| 순위 | 가치 | 설명 |
|------|------|------|
| **1** | **주간 수업 자료** | 코딩/하드코딩 프로그램 + 놀이체육 144개를 **매주 4개씩** 대시보드에서 가이드라인처럼 제공. 구독의 메인 이유. |
| **2** | **출결·학생 데이터 활용** | 저장된 학생/출결로 **술래 정하기, 팀 나누기** 등 수업 전반에 쓰는 간단한 활동 도구. |
| **3** | **리포트·시간 절감** | AI 리포트, 코치 시간 절감은 보조 가치. 리포트는 한도 있으므로 과한 초점 지양. |

---

## 2. 1번: 주간 수업 자료 (대시보드·가이드라인)

### 2.1 자료 구조·노출
- [ ] **대시보드 “이번 주 추천”** 영역: 매주 4개 자료를 주간 가이드라인 형태로 노출
- [ ] 자료 소스 정의: 144개 놀이체육 + (자유도 있는 코딩 프로그램 / 하드코딩 프로그램) 매핑
- [ ] **주차 기준** 명확화: 캘린더 주차 vs 학기 주차 vs “N주차” 라벨 중 어떤 것 쓰는지 결정
- [ ] 구독 플랜별 노출: Free/Basic/Pro에서 “이번 주 4개” 접근 범위 정의 (전부 vs 일부)

### 2.2 콘텐츠·운영
- [ ] 144개 자료 메타데이터 정리: 제목, 난이도, 소요시간, 대상 연령, 키워드(술래/팀 등)
- [ ] “매주 4개” 선정 로직: 수동 큐 vs 자동(랜덤/시즌/테마) vs 하이브리드 결정
- [ ] 대시보드에서 **다운로드/열기/즐겨찾기** 등 액션 가능 여부 및 추적

### 2.3 품질·일관성
- [ ] 주간 가이드라인 UI/카피 톤 통일 (예: “이번 주 수업 가이드”)
- [ ] 자료 누락/404 방지 점검
- [ ] 모바일·태블릿에서 “4개 카드” 가독성 확인

---

## 3. 2번: 출결·학생 데이터로 수업 보조 (술래/팀 등)

### 3.1 팀 나누기
- [ ] **저장된 학생 + 출결** 기반 팀 나누기 (이미 있을 수 있음 → 동작·노출 검증)
- [ ] 기준 옵션: 신체 기능 균형 / 랜덤 / 반·그룹 고정 등
- [ ] 대시보드 또는 “출석부/데이터” 화면에서 1탭 안에 진입 가능한지

### 3.2 술래 정하기
- [ ] 출석한 학생만 후보로 **술래 정하기** (한 명 랜덤 또는 순번)
- [ ] UI: 버튼 한 번에 결과 노출, 반복 사용 가능
- [ ] (선택) “오늘 술래 제외” 등 간단 규칙

### 3.3 기타 수업 전반 활동
- [ ] 리스트: 출석 명단 기반 **순서 정하기 / 그룹 번호 부여 / 역할 배정** 등 후보 정리
- [ ] 우선 1~2개만 구현해 사용성 검증 후 확장

### 3.4 데이터 기반 전제
- [ ] 출결·학생 데이터가 **DB에 안정 저장**됨(Week5 완료) 확인
- [ ] 해당 기능들이 **같은 세션/날짜 기준**으로 일관되게 동작하는지 검증

---

## 4. 3번: 리포트·시간 절감 (보조)

### 4.1 리포트는 “유용하되 한도 내”
- [ ] 플랜별 한도·사용량 노출 명확 (이번 달 N회 남음)
- [ ] **히스토리·재열람**으로 과거 리포트 가치 활용 (이미 구현된 부분 유지)
- [ ] “무제한”에 초점 맞추지 말고, “꼭 필요한 순간에 쓸 수 있음” 메시지

### 4.2 시간 절감 가시화
- [ ] (선택) “이번 주 저장한 시간” 같은 간단 요약이 있다면 문구·위치 점검
- [ ] 리포트보다 **수업 자료 4개 + 술래/팀**이 시간 절감에 기여한다는 스토리 정리

---

## 5. 리텐션·운영 (내실)

### 5.1 핵심 퍼널
- [ ] **로그인 → 대시보드(주간 4개) → 자료 사용** 퍼널 지표 정의
- [ ] **로그인 → 출결/학생 → 술래·팀 나누기** 사용률 추적
- [ ] 주간/월간 “활성 센터” 정의 (주간 4개 1회 이상 열람 등)

### 5.2 결제·구독
- [ ] 결제 실패 시 재시도·알림 (7일/3일/당일)
- [ ] 해지 전 다운그레이드/일시정지 옵션
- [ ] 구독 갱신 시 “주간 자료 계속 받기” 메시지로 1번 가치 강조

### 5.3 Lifecycle 메시징 (선택)
- [ ] Day1: “이번 주 추천 수업 자료 4개 확인하기”
- [ ] Day7: “다음 주 가이드 업데이트” 리마인드
- [ ] 휴면/해지 징후 시 “이번 주 자료 놓치지 마세요” 등

## 6. 실행 순서 제안 (기능 추가 최소·내실 위주)

| 시기 | 1번(자료) | 2번(술래/팀) | 3번·리텐션 |
|------|-----------|----------------|-------------|
| **즉시** | 대시보드 “이번 주 4개” 위치·형식 확정 | 팀 나누기/술래 정하기 현재 구현 여부 파악 | Week5 데이터 영속성·마이그레이션 적용 완료 |
| **1개월** | 매주 4개 선정·노출 로직 고정, 144개 메타 정리 | 술래 정하기 또는 팀 나누기 1개 완성 후 노출 | 결제 실패 복구, 핵심 퍼널 대시보드 |
| **2개월** | 주간 가이드라인 카피/UI 통일, 플랜별 노출 | 두 번째 활동(술래/팀) 추가, 사용처 정리 | D7 리텐션 구간 개선, 해지 사유 수집 |
| **3개월** | “이번 주 자료” 기반 활성 지표로 운영 | 순서/역할 등 1개 더 확장 검토 | 가격/플랜 실험, 월간 성과 요약 |

## 7. 성공 정의 (목적에 맞게)

- **1번 달성**: 구독자 대부분이 **매주 대시보드에서 “이번 주 4개” 수업 자료**를 보고, 수업에 활용한다고 느낀다.
- **2번 달성**: 출결·학생 데이터로 **술래 정하기, 팀 나누기**를 수업 전에 자주 쓴다.
- **3번·리텐션**: 리포트/시간 절감은 보조로 두고, **자료 + 수업 보조 도구**가 구독 유지 이유로 자리 잡는다.
