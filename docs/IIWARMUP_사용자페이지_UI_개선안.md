# iiwarmup 사용자 페이지 UI/UX 개선안

참고: 잘 나가는 피트니스/웜업·교육 앱 UX 패턴을 반영한 구체적 개선 제안.

---

## 참고할 만한 앱/웹 패턴 (2024 기준)

- **한 화면 한 행동**: 첫 화면에서 “지금 할 일”이 하나만 보이게 (예: Nike Training Club, Peloton 앱의 오늘 워크아웃 카드).
- **큰 CTA, 고대비**: 재생/시작 버튼은 화면에서 가장 눈에 띄게, 터치하기 쉽게 (Apple Fitness+, Headspace).
- **진행·완료 피드백**: 프로그레스 바, 단계 표시(1/3), 완료 시 축하 문구/애니메이션으로 이탈 감소 (Future 앱 사례).
- **다크/라이트**: 체육관·실내 환경을 고려한 다크 모드 옵션, 가독성 좋은 폰트 크기.
- **온보딩 최소화**: 60초 이내에 첫 워크아웃에 도달할 수 있게, 단계 수 최소화.

---

## 현재 페이지 대비 구체적 개선안

| 구분 | 현재 | 개선 방향 |
|------|------|-----------|
| **헤더** | 제목 + 한 줄 설명 | 브랜드 강화 + “이번 주 웜업” 같은 한 문장 CTA. 필요 시 히어로 이미지/일러스트 1장. |
| **프로그램 선택** | WeekSelector만 노출 | “2025년 2월 2주차”처럼 읽기 쉬운 라벨, 터치 영역 넓은 주차 선택(카드/칩). |
| **재생 모드** | PhaseControls 버튼들 | **전체 재생**을 메인 CTA(큰 버튼, 강조색), 개별(챌린지/Think/Flow)은 보조 버튼 또는 접기. |
| **빈 상태** | 단순 메시지 | “이번 주 프로그램이 아직 없어요” + 예상 공개일 또는 “이전 주 다시 하기” 옵션. |
| **로딩** | “프로그램 불러오는 중…” | 스켈레톤 또는 프로그레스 인디케이터로 영역 유지. |
| **완료 후** | 곧바로 종료 | “완료!” 문구 + (선택) 홈으로 돌아가기 버튼, 짧은 축하 연출. |
| **접근성** | 기본 수준 | 버튼에 `aria-label`, 포커스 링, 터치 최소 44px. |

---

## 우선 적용 권장 (빠른 효과)

1. **메인 CTA 하나 강조**: “전체 웜업 시작” 같은 단일 메인 버튼 + 나머지는 “챌린지만 / Think만 / Flow만” 보조.
2. **주차 표시 개선**: `WeekSelector`를 “2025년 2월 · 2주차”처럼 읽기 쉽게 + 클릭 영역 확대.
3. **로딩/빈 상태**: 스켈레톤 + 빈 상태 문구/액션 한 줄 정리.
4. **다크 배경 유지 시**: 텍스트 대비 확보(neutral-400 → neutral-300 등), 버튼은 primary 색으로 명확히.

이 문서를 기준으로 [app/iiwarmup/page.tsx](app/iiwarmup/page.tsx) 및 관련 컴포넌트(WeekSelector, PhaseControls)를 단계적으로 수정하면 됨.
